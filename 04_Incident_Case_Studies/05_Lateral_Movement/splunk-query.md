# Lateral Movement via PsExec Pattern — Splunk Detection Queries

## Data Sources

- Index: wineventlog
- Primary events: EventCode 4624 (Windows Security Log) and EventCode 7045 (Windows System Log)
- Target host: DC01 (all queries scoped to the target of lateral movement)

The PsExec-pattern attack generates two distinct events on the target host. Neither event
alone is sufficient for high-confidence detection — Type 3 logons are normal, and service
installations happen regularly. The detection signal is the **correlation**: a privileged
network logon followed by new service creation in a short time window on a sensitive host.


## Query 1 — Service Installation Triage (All Services on DC01)

```spl
index=wineventlog host="DC01" EventCode=7045
| table _time host Service_Name Service_File_Name
| sort -_time
```

![Splunk all-time 7045 query showing SimulatedPsExec at top of results](../screenshots/q1.png)

### What This Shows

Returns 34 total service installations on DC01 all-time. SimulatedPsExec appears at the top
as the most recent entry with `cmd.exe /c echo lateral_movement_test > C:\Temp\lateral_test.txt`
as the binary path. All other entries are legitimate Azure Connected Machine Agent services
installed during lab provisioning on 3/5/26.

This query is the fastest path to spotting the attack — the Service_File_Name column
immediately reveals the anomaly. Legitimate service binary paths reference vendor directories
and named executables. A raw `cmd.exe /c` command in a service binary path has virtually
no legitimate use case.

### What Makes a Service Suspicious

| Indicator | Why It Matters |
|-----------|---------------|
| Binary path is a raw shell command | Legitimate services use dedicated executables |
| Service name has no vendor context | Real services follow naming conventions |
| Installed during an active user session | System services install at provisioning time |
| Service account is LocalSystem | High-privilege services warrant extra scrutiny |
| Binary path points to temp/user directories | Staged malware commonly uses C:\Temp, %APPDATA% |


## Query 2 — Targeted Event Collection (Administrator Logons + Services)

```spl
index=wineventlog host="DC01" ((EventCode=4624 AND Logon_Account_Name=Administrator) OR EventCode=7045)
| table _time host EventCode Logon_Account_Name Logon_Type Service_Name Service_File_Name
| sort -_time
```

![Splunk query showing 6 events — Administrator Type 3 logons surrounding SimulatedPsExec 7045](../screenshots/q2.png)

### What This Shows

Returns 6 events across a 5-minute window scoped to the attack. The 4624 at 21:14:39 is
the initial ADMIN$ connection from CLIENT01. The 7045 at 21:16:04 is the service creation.
The cluster of 4624 events after 21:16:04 are generated by the sc.exe start and control
channel re-authentications — each remote sc.exe operation generates its own network logon.

### Blank Columns Are Expected

The table intentionally mixes two event types that don't share all fields:
- 4624 rows populate `Logon_Account_Name` and `Logon_Type` — `Service_Name` and `Service_File_Name` are blank
- 7045 rows populate `Service_Name` and `Service_File_Name` — logon fields are blank

This is correct behaviour. The blank cells visually separate the two event types and make
the attack pattern readable — logon rows bookend the service creation row.


## Query 3 — Full Attack Chain Correlation

```spl
index=wineventlog host="DC01"
| where EventCode=4624 AND Logon_Type=3
| append [search index=wineventlog host="DC01" EventCode=7045]
| sort _time
| table _time EventCode Logon_Account_Name Logon_Type Service_Name Service_File_Name
```

![Splunk correlation query showing 61 events with attack chain visible on page 2](../screenshots/q3.png)

### What Each Line Does

| Line | Purpose |
|------|---------|
| `where EventCode=4624 AND Logon_Type=3` | Filter to network logons only — eliminates interactive, service, and batch logon noise |
| `append [search ... EventCode=7045]` | Merge service creation events into the same result set |
| `sort _time` | Chronological order — attack chain reads naturally top to bottom |
| `table ...` | Combined view: logon fields and service fields side by side |

### Why `append` Instead of a Join

`append` adds rows to the result set without requiring a matching key. This is correct here
because 4624 and 7045 events share no common linking field — they come from different log
sources (Security vs System log) and have no session ID in common. The correlation is
temporal: proximity in time on the same host is the detection signal.

### Attack Chain Visible in Results

The attack sequence is readable on page 2 of the 61-event result set:

```
21:14:39  4624  Administrator  3  —  —              ← Initial ADMIN$ connection from CLIENT01
21:16:04  4624  Administrator  3  —  —              ← sc.exe create authentication
21:16:04  7045  —  —  SimulatedPsExec  cmd.exe /c echo lateral_movement_test > C:\Temp\lateral_test.txt
21:16:12  4624  asmith  3  —  —                     ← Separate account — investigate
21:16:14  4624  Administrator  3  —  —              ← sc.exe start authentication (x3)
```

The 85-second gap between logon and service creation is consistent with scripted or
tool-assisted attack execution. The presence of `asmith` Type 3 logons in the same window
is a separate finding requiring investigation.


## Query 4 — Broad Lateral Movement Hunt (Domain-Wide)

```spl
index=wineventlog EventCode=7045
| search NOT (Service_File_Name="*Program Files*" OR Service_File_Name="*Windows*" OR Service_File_Name="*system32*")
| table _time host Service_Name Service_File_Name
| sort -_time
```

Hunts for suspicious service installations across all hosts by excluding known-good
Windows and Program Files paths. Use after an initial detection to determine if the
attacker moved laterally to additional hosts beyond DC01.


## Query 5 — LOLBin Service Binary Path Detection

```spl
index=wineventlog EventCode=7045
| where match(Service_File_Name, "(?i)cmd\.exe|powershell|wscript|cscript|mshta|rundll32|regsvr32")
| table _time host Service_Name Service_File_Name
| sort -_time
```

Targets the specific pattern of PsExec-style attacks — a service whose binary path is a
shell interpreter or LOLBin rather than a dedicated service executable. Legitimate services
almost never use cmd.exe or PowerShell as their primary binary.

| Binary | Why It's Suspicious in a Service Path |
|--------|--------------------------------------|
| cmd.exe | Shell — exits immediately, not a persistent service |
| powershell.exe | Script host — common C2 execution vehicle |
| wscript.exe / cscript.exe | Script engines — scriptlet payloads |
| mshta.exe | HTML Application host — remote script execution |
| rundll32.exe | DLL execution — fileless payload delivery |
| regsvr32.exe | COM registration — script execution bypass |


## Raw Event Reference

### Event ID 4624 — Type 3 Network Logon

Key fields for lateral movement detection:

| Field | Attack Value | Why It Matters |
|-------|-------------|---------------|
| Logon Type | 3 | Network — SMB/ADMIN$ connection |
| Account Name (New Logon) | Administrator | Privileged account used remotely |
| Security ID | S-1-5-21-...-500 | Built-in Administrator — SID pivot |
| Logon ID | 0xF04D084 | Session key — correlate all activity in this session |
| Elevated Token | Yes | Full admin rights in session |
| Impersonation Level | Delegation | Can impersonate on further remote systems |
| Source Network Address | 192.168.113.20 | CLIENT01 — attack origin confirmed |
| Logon Process | Kerberos | Domain authentication |

### Event ID 7045 — New Service Installed

Key fields for lateral movement detection:

| Field | Attack Value | Why It Matters |
|-------|-------------|---------------|
| Service Name | SimulatedPsExec | Non-standard, no vendor context |
| Service File Name | cmd.exe /c echo lateral_movement_test > C:\Temp\lateral_test.txt | Raw command — not an executable |
| Service Account | LocalSystem | Maximum local privilege |
| Service Start Type | Demand start | Manually triggered, not persistent |
| SID | S-1-5-21-...-500 | Matches Administrator from 4624 — same session |

## Logon Type Reference

| Type | Name | Common Source |
|------|------|--------------|
| 2 | Interactive | Direct keyboard login |
| 3 | Network | net use, SMB, sc.exe remote |
| 4 | Batch | Scheduled tasks |
| 5 | Service | Service account startup |
| 10 | RemoteInteractive | RDP |
| 11 | CachedInteractive | Cached credential login |

## False Positive Considerations

| Source | Pattern | How to Distinguish |
|--------|---------|-------------------|
| Legitimate deployment tools | Push software via SMB, install as service | Known service names, signed binary paths, scheduled timing |
| Group Policy processing | Network logons to DCs | Source is SYSTEM account, not named admin |
| Backup agents | Mount shares, write service components | Known vendor paths in Service_File_Name |
| Vulnerability scanners | Network logons only | Generate 4624 but not 7045 |

When 4624 Type 3 is immediately followed by 7045 with an unrecognised service name and a
non-standard binary path on a domain controller, the combination is high-confidence for
immediate investigation.
